HTML Forms: a Cheatsheet

Types of <input>:

Name Description Syntax
Checkbox Renders a single checkbox item. Related checkboxes must share the same name attribute type="checkbox"
Number Renders a text field, in which only numbers, '.', '+', '-' can be entered type="number"
Range Renders a slider. Important attributes: min, max, step type="range"
Text Renders a single-row text field type="text"
Radio Renders a single radio button. If multiple radio buttons are assigned the same name, only one of these buttons can be selected at a time. type="radio"
Password Renders a single-row text field, where user input is censored type="password"
Submit Renders a submission button. Default text of "Submit" can be changed by modifying the value attribute. type="submit"

Attributes of <input>:

Attribute Description
type Affects the way the element renders on the page plus the type of input allowed
name This is a very important attribute, as a pair of name and value is generally what will be sent upon submission of a form.
id This is mainly useful for connecting input elements with label elements, which should have the input's id as their for attribute

Alternatives to <input>:

Tag Description Important attributes
<textarea> Creates a multi-line textbox for input. Unlike <input>, it must be paired with a closing tag. Text between the two tags will become placeholder text that starts in the field. rows, cols, value, form, name
<select> Creates a dropdown list for selection. Only one choice, corresponding to one <option> element, can be selected at a time. name. For <option>: value
<datalist> This is not so much an alternative to the input element as it is a special variety. If paired with the <input> element, it will render a text field with a searchable list of pre-defined options, while maintaining the user's ability to write their own. The input's list attribute must match the datalist's id id. For <input>: list. For <option>: value
