The <input> Element: an HTML Cheatsheet

Struggling to understand HTML forms?

Overwhelmed by all the different types of <input>?

You've come to the right place.

Types of <input>:

Name Description Syntax
Checkbox Renders a single checkbox item. Related checkboxes must share the same name attribute. type="checkbox"
Number Renders a text field, in which only numbers and number-related characters can be entered. type="number"
Range Renders a slider. Important attributes: min, max, step. type="range"
Text Renders a single-row text field. type="text"
Radio Renders a single radio button. If multiple radio buttons are assigned the same name, only one of these buttons can be selected at a time. type="radio"
Password Renders a single-row text field, where user input is censored. type="password"
Submit Renders a submission button. Default text of "Submit" can be changed by modifying the value attribute. type="submit"

Attributes of <input>:

Attribute Description
type Affects the way the element renders on the page plus the type of input allowed. See above table.
value When a form is submitted, the user's input is sent in the form of pairs of name and value. For a text input, value would be what is in the text box. If manually assigned, it will set default input for a form.
name This is a very important attribute, as pairs of name and value will be sent upon submission of a form.
id This is mainly useful for connecting input elements with label elements, which should have the input's id as their for attribute.
required This straightforward attribute doesn't need to be set to anything; if present, it means that the field must be filled out in order to submit the form. Can also be written as required="true".
min When used with an <input> of type="number" or type="range", this attribute dictates the minimum value.
max When used with an <input> of type="number" or type="range", this attribute dictates the maximum value.
minlength When used with text-based <input> fields, this attribute dictates the minimum number of characters for the field.
maxlength When used with text-based <input> fields, this attribute dictates the maximum number of characters for the field.
pattern When used with text-based <input> fields, checks the chosen regular expression against the input upon form submission. E.g. pattern="[A-Za-z]{3}" means that the user must input three letters between A and Z; failing to do so will display an error message.

Alternatives to <input>:

Tag Description Important attributes
<textarea> Creates a multi-line textbox for input. Unlike <input>, it must be paired with a closing tag. Text between the two tags will become placeholder text that starts in the field. rows, cols, value, name, form
<select> Creates a dropdown list for selection. Only one choice, corresponding to one <option> element, can be selected at a time. For <select>: name.
For <option>: value
<datalist> This is not so much an alternative to the input element as it is a special variety. If paired with the <input> element, it will render a text field with a searchable list of pre-defined options, while maintaining the user's ability to write their own. The input's list attribute must match the datalist's id. For <datalist>: id.
For <input>: list.
For <option>: value
